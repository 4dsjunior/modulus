2. Arquitetura Front-end (HTML/JavaScript)

O projeto é construído em um único arquivo HTML utilizando JavaScript Vanilla para a lógica e Tailwind CSS (via CDN) para o estilo, garantindo alta performance e portabilidade.

2.1. Estrutura de Layout (UX/UI)

KPIs (Key Performance Indicators):

Exibidos em cards brancos (#FFFFFF) com sombras sutis (shadow-kpi).

Bordas laterais coloridas para categorização: Primária (Acumulado), Acento (Previsão) e Sucesso (Alunos Ativos).

Status Mensal:

Painel expandido que mostra a diferença entre Esperado, Recebido e Em Falta.

Usa tamanhos de fonte grandes (2rem) para os valores, garantindo impacto imediato.

Tabela de Pagamentos Pendentes:

Design zebrado (linhas alternadas em #F8F9FA) para melhorar a leitura em linhas longas.

Valores pendentes em cor de Alerta (#FF5722) e botão de ação em Laranja Acento (#FF9800).

Segmentação de Alunos Ativos:

Apresenta os dados em formato de lista hierárquica e um gráfico interativo (Doughnut Chart).

Correção de UX/UI: Os ícones de gênero foram substituídos pelas letras M e F para garantir consistência e legibilidade em todos os dispositivos.

2.2. Lógica de Dados (Mock e Funções Principais)

Todos os dados são simulados em memória (Mock Data) para demonstrar a funcionalidade sem dependência de um backend real.

a) Estado Global (MOCK)

let mockPendingPayments = []; // Pagamentos aguardando aprovação
let mockStats = { approvedPayments: [], students: [] }; // Alunos e histórico de pagamentos
let modalityFrequencyCounts = {}; // Estrutura de contagem de alunos por Modalidade/Frequência
let modalityRevenueCounts = {}; // Faturamento potencial por Modalidade


b) Funções de Renderização Dinâmica

renderModalityFrequency(data): Constrói a lista hierárquica de alunos ativos.

updateChart(id, type, labels, data, ...): Utiliza a biblioteca Chart.js para criar gráficos de rosca (Doughnut) dinamicamente, alternando entre Contagem de Alunos e Faturamento Potencial (dependendo da interação do usuário).

renderPrimaryModalityChart(): Modo padrão do gráfico, focado no Faturamento Potencial por Modalidade.

renderFrequencyChart(modality): Ativado por clique na lista, quebra a contagem de alunos por frequência dentro da modalidade selecionada.

c) Lógica de Negócios (Simulada)

approvePayment(paymentId, ...): Remove o pagamento da lista mockPendingPayments, adiciona ao mockStats.approvedPayments e simula a atualização da due_date do aluno para +30 dias.

fetchAndRenderStatistics(): Recalcula todas as KPIs (Faturamento Anual, Recebido Mensal, Previsto, Alunos Ativos) e atualiza a segmentação a cada alteração de estado.

registerNewStudent(formData): Adiciona um novo aluno à lista mockStats.students e recalcula as estatísticas.

d) Modais de UX

O projeto utiliza modais customizados (#confirmation-modal e #registration-modal) em vez das funções nativas alert() e confirm(), garantindo uma experiência de usuário consistente e não-bloqueante.
